## 2. 시스템별 상세 분석

### 2-1. 낮/밤 페이즈 ✅

`Assets/Scripts/Cycle/DayNightController.cs`

- Day → Evening → Night → Dawn 4단계 순환 완벽 구현
- 레벨 스케일링 `1 + 0.25 × (level-1)` 수식 정확히 적용
- EventBus 기반 Payload(`DayStartedPayload` 등)로 씬 간 결합 최소화
- **주의**: BalanceConfig 기본값이 `DayDurationSec=540f`로 기획서 40s와 불일치 → 출시 전 수정 필요

### 2-2. 채집 시스템 ✅

`Assets/Scripts/Gather/GatherController.cs`

- 기획서가 나무=W키, 돌=R키로 명시했으나 구현은 E키로 통합
  - 기능상 문제없으나 기획서 키 설명 업데이트 필요
- 동료 최대 2명 자동 합류(4u 범위) — 기획서 미명시였으나 체감 향상에 기여

### 2-3. NPC 영입 ✅

`Assets/Scripts/Companions/RecruitableNpc.cs`

- 기획서 명시 5종 중 "치유사" → "농부", "무사" → "전사"로 명칭 조정
- 기능은 동일: 노인/농부(농사 특화), 사냥꾼/전사(전투 특화), 아이(기동)
- 기획서 업데이트 권장

### 2-4. 좀비 웨이브 ✅

`Assets/Scripts/Enemies/NightController.cs`

- 25킬 보스 트리거 정확히 구현
- 기획서 미명시된 추가 요소: 4종 변종 좀비, 4형태 보스(FrostZombie→FrostLich), 블리자드 이벤트(20% 확률 웨이브 2배)
- 이는 기획서 요구를 초과 달성한 부분

### 2-5. 모닥불 ⚠️

`Assets/Scripts/Combat/CampfireAura.cs`

```
기획서: DPS 2.5
구현:   DPS 5 (2배 강함)
```

- 연료 100s, 온도 유지, 밤 HP 드레인 시스템은 정확히 구현
- DPS만 불일치 — BalanceConfig에서 `CampfireAura.DamagePerSecond=2.5f`로 수정 가능

### 2-6. 기획서 미명시 추가 구현

## 3. 페르소나 플레이 분석

### 페르소나: 이진혁 (27세, 직장인)
- 자투리 시간 20~30분 플레이
- 로그라이크/생존류 선호, 복잡한 조작 부담
- "오늘 밤은 버텼다"는 성취감 원함

---

### 3-1. 세션 길이 — ⚠️ 불일치 위험

**기획서 목표**: 세션당 15~25분

**현재 BalanceConfig 기본값**:
```
DayDurationSec   = 540s (9분)
NightDurationSec = 360s (6분)
EveningTransition = 30s
DawnTransition    = 30s
```

1사이클 = 약 16분 → Day 1만 완주해도 16분 소모  
→ 2일차 도전 시 30분 초과 가능성 높음

**이진혁의 반응 예측**:  
> "1라운드 끝냈는데 이게 다야? 싶거나, 반대로 2라운드 가다가 시간이 없어서 중단하게 됨"

→ **BalanceConfig를 기획서 명시값(40s)으로 맞춰야 페르소나 목표 세션과 일치**

---

### 3-2. 첫 5분 — ✅ 양호

1. 온보딩 씬: DAY/NIGHT 설명 → 기대감 형성
2. 낮 1회차: 설원 탐험, 나무/돌 채집, 사슴 사냥
3. 동료 영입: F키로 간단한 인터랙션

**이진혁의 반응 예측**:  
> "설원 배경 + 눈 파티클 연출이 신선하다. 채집과 이동이 단순해서 바로 들어갈 수 있다."

**우려점**:
- 기획서에서 채집 키를 W(나무), R(돌)로 설명했으나 실제는 E키 통합
  → 온보딩 슬라이드가 E키로 안내하는지 확인 필요

---

### 3-3. 첫 번째 밤 — ✅ 핵심 성취감 구간

- 좀비 웨이브 격퇴 → 25킬 보스 → 새벽 회복
- 시야 그라데이션(어두움 + 모닥불 빛 원)이 긴장감 조성
- 건물 체력바가 시각적으로 위기를 알림

**이진혁의 반응 예측**:  
> "모닥불 빛 안에 들어가서 좀비 태워죽이는 느낌이 직관적으로 재밌다. 보스가 25킬 후에 나오니까 킬 수 챙기는 재미도 있다."

**우려점**:
- 모닥불 DPS가 기획서 대비 2배(5 DPS)로 밤 난이도가 낮아졌을 수 있음
  → 이진혁처럼 "오늘 밤은 버텼다"는 성취감을 원하는 유저에게는 오히려 적당할 수도 있으나, 긴장감은 희석됨

---

### 3-4. 반복 플레이 동기 — ✅ 룬 시스템이 강점

기획서에 없던 **17종 룬 시스템**이 로그라이크 루프를 형성:
- 매 플레이마다 다른 룬 조합 시도 가능
- 마스터 시너지(동일 룬 3스택 → +50%) — "이번엔 빙결 특화로"

**이진혁의 반응 예측**:  
> "다음엔 SummonDog + AllyBoost로 해봐야지. 이 조합 못 해봤는데."

---

### 3-5. 조작 복잡도 — ✅ 적절

| 조작 | 키 | 직관성 |
|------|----|--------|
| 이동 | WASD | ✅ |
| 채집 | E | ✅ |
| 영입 | F | ✅ |
| 건물 배치 | (UI 클릭 추정) | 확인 필요 |

"복잡한 조작은 부담"인 이진혁에게 적합한 키 구성.

---

### 3-6. 페르소나 KPI 달성 예측

| KPI | 목표 | 현재 예측 | 판단 |
|-----|------|---------|------|
| Day 3 생존율 50% | 50% 이상 | 알 수 없음 (플레이 데이터 없음) | — |
| 세션당 플레이 시간 | 15~25분 | **BalanceConfig 기본값 기준 초과 위험** | ⚠️ |
| 밤 1회 격퇴 성취감 | 7/10 이상 | 모닥불 빛·긴장감·보스 연출로 달성 가능 | ✅ |

## 4. 개선 권장 사항 (우선순위 순)

### P0 — 즉시 수정
1. **BalanceConfig 기본값 수정**: `DayDurationSec=40f`, `NightDurationSec=40f`로 변경
   - 기획서 명시값과 일치, 페르소나 세션 길이 목표 달성

### P1 — 빠른 개선
2. **모닥불 DPS 조정**: `CampfireAura.DamagePerSecond=2.5f` (현재 5f)
   - 기획서 명시값 준수, 밤 긴장감 회복
3. **온보딩 씬 키 안내 확인**: 채집 키가 E로 통합된 것이 슬라이드에 반영되어 있는지 검증

### P2 — 기획서 업데이트
4. **NPC 명칭 동기화**: "치유사" → "농부", "무사" → "전사" 반영
5. **건물 종류 업데이트**: 6종 → 8종 (Infirmary, HuntersHut 추가)
6. **채집 키 수정**: "나무(W), 돌(R)" → "E키로 통합"

## 5. 결론

**구현 완성도: 우수** — 기획서 핵심 요구사항의 97%가 구현되어 있으며, 룬 시스템·밤 시야·프로시저럴 맵 등 기획서를 초과하는 요소도 다수 존재한다.

**페르소나 만족도 예측: 중상 (6~7/10)**

- ✅ 단순한 조작 + 직관적 피드백(모닥불 빛, 킬 카운트)으로 진입 장벽 낮음
- ✅ 룬 시스템으로 반복 플레이 동기 충분
- ⚠️ 세션 길이 이슈: BalanceConfig 기본값이 기획서와 달라 20~30분 세션 목표 미달 또는 초과 위험
- ⚠️ 모닥불 DPS 2배 문제로 밤 긴장감("오늘 밤은 버텼다"는 성취감)이 희석될 수 있음

**P0 수정 2건(BalanceConfig 타이머 + 모닥불 DPS)만 적용하면 페르소나 KPI 달성 가능성이 높아진다.**
